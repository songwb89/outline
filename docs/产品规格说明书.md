# 教育管理系统 - 产品规格说明书

## 一、产品概述

### 1.1 产品定位

面向教育局和学校的管理系统，支持教育局统一采购和学校独立采购两种模式。

### 1.2 核心概念

- **组织单元**：树形层级结构，可表示省、市、区县、教育局、区域等
- **学校**：挂在组织单元下的最终使用单位
- **功能模块**：系统中的各个功能（如：阅卷、成绩分析、报表等）
- **套餐**：包含多个功能模块 + 初始点数的组合包
- **点数**：用于消耗型功能，每次使用扣减点数
- **充值**：套餐内点数用完后，单独购买点数

---

## 四、核心业务流程

### 4.1 套餐设计流程（管理员）

```
1. 在"功能模块管理"中新增功能模块
   - 设置模块名称、类型（消耗点数/无限使用）
   - 设置消耗型模块的单次扣点数
2. 在"套餐管理"中新增套餐
   - 设置套餐名称、价格、有效期
   - 设置初始点数
   - 勾选套餐包含的功能模块，设置各模块包含次数
3. 套餐创建完成，可用于采购
```

### 4.2 采购流程

```
1. 在"学校管理"中创建学校
2. 在"采购管理 > 开套餐"中新增采购记录
   - 选择学校
   - 选择套餐（自动带入初始点数）
   - 填写成交金额和日期
3. 采购生效，学校获得：
   - 套餐包含的功能权限
   - 对应的初始点数
```

### 4.3 充值流程（点数用完时）

```
1. 运营人员发现学校点数不足或用完
2. 进入学校详情页（学校管理 > 点数管理）
3. 点击"充值"按钮
   - 选择该学校的采购记录
   - 填写充值点数和金额
4. 充值成功，学校可用点数增加
```

### 4.4 试用开通流程

```
1. 在"采购管理 > 试用开通"中新增试用记录
   - 选择学校
   - 选择试用套餐
   - 设置试用点数和天数
2. 试用生效，学校获得试用功能
3. 试用到期后，可选择：
   - 转正为正式采购（试用剩余点数累加）
   - 结束试用
```

### 4.5 点数消耗流程（系统自动）

```
1. 学校使用某个消耗型功能（如智能阅卷）
2. 系统检查：
   - 学校是否在有效期内
   - 学校剩余点数是否充足
3. 满足条件则扣减点数，记录消费
4. 不满足则提示：点数不足，请联系管理员充值
```

## 六、备注

1. **套餐与点数的关系**：

   - 套餐包含初始点数和功能权限
   - 采购后，学校获得套餐设定的初始点数
   - 使用消耗型功能时，扣减点数
   - 点数用完后，可单独充值

3. **导出功能**：各列表页面支持导出 Excel
