# 套餐权限

### 1.1 项目背景与目标

**项目背景**

为解决当前系统缺乏灵活套餐体系和精细权限控制的问题，亟需构建一套完整的套餐权限系统。该系统需支持套餐配置、点数消耗与追踪，以及订单全生命周期管理，从而满足平台运营方与学校用户的不同需求。

**项目目标**

- 建立灵活的套餐管理体系，支持功能模块化组合、接口试用配置，满足不同学校用户的差异化需求
- 支持学校管理员独立维护本校教师额度
- 构建点数消费体系，实时追踪接口调用点数消耗，提供清晰的点数变动明细
- 实现订单全流程管理，覆盖试用、开通、充值、续费等多种订单类型
- 支持配置计费接口与权限菜单，灵活控制功能可见性

## 二、功能模块需求

### 3.1 套餐管理

#### 3.1.1 套餐列表

**数据展示**

| 字段     | 说明                                   |
| -------- | -------------------------------------- |
| 套餐名称 | 套餐的名称和描述                       |
| 类型     | 计费版 / 试用版                        |
| 套餐额度 | 套餐内的点数                           |
| 教师上限 | 可创建的教师账号数量，不限则显示"不限" |
| 有效期   | 套餐有效天数                           |
| 价格     | 套餐售价                               |
| 状态     | 上架 / 下架                            |
| 操作     | 查看、编辑、删除                       |

**查询与排序**

- 默认按创建时间降序排列
- 支持分页展示，每页10条数据

**状态说明**

- 上架：学校在开通套餐时可选择该套餐
- 下架：无法选择该套餐，仅已开通的学校可继续使用

#### 3.1.2 新增/编辑套餐

**套餐类型**

套餐分为"计费版"和"试用版"两种类型，类型仅作为标记展示，不影响表单内容逻辑。

**字段规格**

| 字段         | 规格                               |
| ------------ | ---------------------------------- |
| 套餐类型     | 必选（新增时选择，编辑时不可修改） |
| 套餐名称     | 必填，不超过20字                   |
| 有效期       | 必填，正整数天数                   |
| 套餐额度     | 必填，非负整数点数                 |
| 教师账号上限 | 选填，正整数；留空不限制           |
| 价格         | 必填，非负数（元）                 |
| 套餐描述     | 选填，简要描述套餐内容             |
| 开放功能     | 必选，至少选择一个                 |
| 免费次数配置 | 选填，非负整数，默认为0次          |
| 状态         | 上架/下架，默认上架                |

### 3.2 学校管理

#### 3.2.1 学校列表

**数据展示**

| 字段     | 说明                                            |
| -------- | ----------------------------------------------- |
| 学校名称 | 学校名称                                        |
| 学校编码 | 学校编码                                        |
| 账号     | 管理员登录账号                                  |
| 联系人   | 联系人姓名                                      |
| 套餐     | 当前套餐名称 + 有效期；若已过期则显示"过期"标签 |
| 充值点数 | 该学校累计充值点数                              |
| 已消耗   | 已消耗的点数                                    |
| 剩余点数 | 当前剩余可用点数                                |
| 账号状态 | 正常 / 禁用                                     |
| 操作     | 编辑、管理、续费（套餐为计费版才显示此操作）    |

**查询条件**

- 学校名称：按学校名称模糊搜索
- 账号：按学校管理账号搜索
- 联系人：按联系人姓名模糊搜索
- 账号状态：全部 / 可用 / 禁用
- 套餐状态：全部 / 有效 / 过期

**状态说明**

- **账号状态**

  - 正常：学校的账号（管理账号、教师账号）可正常登录
  - 禁用：学校的所有账号禁止登录
- **套餐状态**

  - 有效：套餐在有效期内，套餐内的功能权限可用
  - 过期：套餐已到期，套餐内的功能权限不可用

#### 3.2.2 新建/编辑学校

**表单字段**

| 字段     | 类型     | 必填                           | 说明                                             |
| -------- | -------- | ------------------------------ | ------------------------------------------------ |
| 学校名称 | 文本     | 是                             | 输入学校全称；新建时需去重                       |
| 学校编码 | 文本     | 是                             | 2~6位字母或数字，新建时需去重；编辑时不可改      |
| 账号     | 组合输入 | 是                             | 6~16位字母或数字；新建时需去重；编辑时不可改     |
| 密码     | 文本     | 新建时必填，<br />编辑时非必填 | 新建时必填；编辑时留空不修改，填写则更新为新密码 |
| 联系人   | 文本     | 否                             | 姓名或职位                                       |
| 手机号   | 文本     | 否                             | 最多11位                                         |
| 状态     | 下拉选择 | 否                             | 可用（默认）/ 禁用                               |

#### 3.2.3 开通试用

**前置条件**：学校当前无有效期内的计费套餐。

**校验逻辑**：

- 检查学校是否存在有效期内的计费套餐
- 若存在 → Toast 提示"该学校存在套餐，无法开通试用"，阻止操作

**通过后操作**：

- 新增一条"试用"类型订单，记录开通时间、试用期限等信息
- 更新学校的套餐为试用套餐
- 更新学校套餐的有效期（从当前日期开始，截止日期为当天 + 试用期限天数）
- 若套餐包含点数（> 0），在学校点数流水表中新增一条类型为"充值"的记录，备注为"套餐充值"。

**弹窗字段**

| 字段     | 类型     | 必填 | 说明                   |
| -------- | -------- | ---- | ---------------------- |
| 学校名称 | 选择器   | 是   | 所有学校               |
| 套餐     | 选择器   | 是   | 显示已启用的计费套餐   |
| 成交金额 | 数字输入 | 是   | 输入成交金额，数字类型 |

#### 3.2.4 开通套餐

**前置条件**：为学校选择一个具体的计费套餐。

**校验逻辑**：

- 检查学校是否存在相同套餐（不考虑是否过期）
- 结果分为两种情况

**存在相同套餐**：

- 订单类型为"续费"
- 学校当前套餐不变（不替换）
- 若原套餐已过期：有效期从今天开始，截止日期 = 今天 + 套餐天数
- 若原套餐未过期：原截止日期 + 套餐天数，起始日期不变
- 新增一条"续费"订单
- 若套餐包含点数（> 0），在学校点数流水表中新增一条类型为"充值"的记录，备注为"套餐充值"

**不存在该套餐**：

- 订单类型为"开通"
- 学校当前套餐变更为新套餐
- 有效期从今天开始，截止日期 = 今天 + 套餐天数
- 新增一条"开通"订单
- 若套餐包含点数（> 0），在学校点数流水表中新增一条类型为"充值"的记录，备注为"套餐充值"

**弹窗字段**

| 字段     | 类型     | 必填 | 说明                   |
| -------- | -------- | ---- | ---------------------- |
| 学校名称 | 选择器   | 是   | 所有学校               |
| 套餐     | 选择器   | 是   | 显示已启用的计费套餐   |
| 成交金额 | 数字输入 | 是   | 输入成交金额，数字类型 |

#### 3.2.5 续费

**适用场景**：学校已有套餐（计费套餐），需要延长有效期。

**前提条件**：学校必须有已开通的计费套餐。

**操作逻辑**：

- 订单类型为"续费"
- 学校当前套餐不变
- 若原套餐已过期：有效期从今天开始，截止日期 = 今天 + 套餐天数
- 若原套餐未过期：原截止日期 + 套餐天数，起始日期不变
- 新增一条"续费"订单
- 若套餐包含点数（> 0），在学校点数流水表中新增一条类型为"充值"的记录，备注为"套餐充值"

**弹窗字段**

| 字段     | 说明                           |
| -------- | ------------------------------ |
| 学校名称 | 显示当前操作的学校名称（只读） |
| 套餐     | 显示学校当前套餐（只读）       |
| 成交金额 | 输入成交金额，数字类型。必填。 |

#### 3.2.6 点数明细

**入口**：在学校管理列表中，点击某个学校的"管理"按钮，打开点数明细弹窗。

**弹窗结构**

**1. 标题区**

| 字段   | 说明                     |
| ------ | ------------------------ |
| 标题   | 学校名称 + " - 点数明细" |
| 副标题 | 学校编码                 |

**2. 汇总卡片**（顶部三个统计卡片）

| 指标     | 说明                              |
| -------- | --------------------------------- |
| 充值点数 | 该学校累计充值的点数总和          |
| 已消耗   | 该学校累计消费+扣减的点数总和     |
| 剩余点数 | 当前可用点数（充值 - 消费-扣减） |

**3. 数据表格**

| 列名   | 说明                                               |
| ------ | -------------------------------------------------- |
| 类型   | 标签样式：充值（绿色）/ 消费（蓝色）/ 扣减（橙色） |
| 点数   | 充值显示（+N）；消费/扣减显示（-N）                |
| 时间   | 精确到分钟，格式：YYYY-MM-DD HH:mm                 |
| 操作人 | 执行该操作的用户名称和账号                         |
| 备注   | 操作说明，无备注显示"-"                            |

#### 3.2.7 充值点数

**入口**：在点数明细弹窗中，点击"充值"按钮，打开充值弹窗。

**弹窗字段**

| 字段 | 类型     | 必填 | 说明                               |
| ---- | -------- | ---- | ---------------------------------- |
| 点数 | 数字输入 | 是   | 输入要充值的点数，最小值为 1       |
| 金额 | 数字输入 | 是   | 输入充值金额，单位：元，最小值为 0 |
| 备注 | 多行文本 | 否   | 选填，用于记录充值说明             |

**确认后操作**

1. 在学校点数流水表中新增一条类型为"充值"的记录：

   - 点数 = 输入的点数（正值）
   - 备注 = 填写的备注内容
   - 操作人 = 当前操作人员
   - 时间 = 提交时间
2. 在订单管理中新增一条类型为"充值"的订单记录。

   - 金额 = 输入的金额
   - 点数 = 输入的点数（正值）
   - 备注 = 填写的备注内容
   - 创建人 = 当前操作人员

#### 3.2.8 扣减点数

**入口**：在点数明细弹窗中，点击"扣减"按钮，打开扣减弹窗。

**弹窗字段**

| 字段     | 类型     | 必填 | 说明                         |
| -------- | -------- | ---- | ---------------------------- |
| 点数     | 数字输入 | 是   | 输入要扣减的点数，最小值为 1 |
| 扣减原因 | 多行文本 | 是   | 必填，用于记录扣减说明       |

**校验规则**

- 点数：必填，最小值为 1，不满足则提示"请输入有效的点数"
- 扣减原因：必填，不满足则提示"请输入扣减原因"

**确认后操作**

1. 在学校点数流水表中新增一条类型为"扣减"的记录：

   - 点数 = 输入的点数
   - 备注 = 填写的扣减原因
   - 操作人 = 当前操作人员
   - 时间 = 提交时间

### 3.3 订单管理

**说明**：订单数据由学校管理模块的操作自动生成，包括开通试用、开通套餐、续费、充值四种类型。

#### 3.3.1 订单列表

**筛选条件**

| 字段 | 类型     | 说明                             |
| ---- | -------- | -------------------------------- |
| 类型 | 下拉选择 | 全部 / 试用 / 开通 / 充值 / 续费 |
| 学校 | 文本输入 | 按学校名称模糊搜索               |
| 套餐 | 下拉选择 | 按套餐名称筛选                   |

**数据表格**

| 列名     | 说明                                                                                 |
| -------- | ------------------------------------------------------------------------------------ |
| 订单号   | 系统自动生成，格式：ORD + 日期（YYYYMMDD）+ 序号                                     |
| 类型     | 标签样式：试用（紫色）/ 开通（蓝色）/ 充值（绿色）/ 续费（橙色）                     |
| 学校名称 | 学校名称（学校编码）                                                                 |

**订单类型说明**

| 类型 | 触发场景 | 套餐列             | 备注列                   |
| ---- | -------- | ------------------ | ------------------------ |
| 试用 | 开通试用 | 显示试用套餐名称   | 显示起止日期             |
| 开通 | 开通套餐 | 显示开通的套餐名称 | 显示起止日期             |
| 续费 | 套餐续费 | 显示续费的套餐名称 | 显示起止日期             |
| 充值 | 充值     | 显示 "--"          | 显示充值时填写的备注信息 |

### 3.4 扣点规则

**说明**：配置接口的扣点规则，教师端调用接口时根据规则扣减点数。

#### 3.4.1 接口分组

- 支持新增分组，配置分组名称
- 可编辑分组名称
- 可删除分组（删除时一并删除该分组下的接口配置）

#### 3.4.2 接口配置

每个接口包含以下字段：

| 字段     | 说明               |
| -------- | ------------------ |
| 接口名称 | 接口的中文名称     |
| 接口路径 | 接口的请求路径     |
| 计费点数 | 每次调用扣减的点数 |

**使用场景**：教师端调用接口时，系统根据配置的接口路径匹配规则，自动扣减相应点数。

### 3.5 菜单管理

#### 3.5.1 页面配置（一级）

- 支持新增页面，配置页面名称和页面编码
- 可编辑页面信息
- 支持开关状态：关闭后教师端所有用户都看不到该页面（不是针对某个套餐隐藏）
- 可删除页面（删除时一并删除该页面下的功能按钮）

#### 3.5.2 功能按钮（二级）

- 在页面下新增功能按钮，配置按钮名称、编码和排序号
- 可编辑功能按钮信息
- 支持开关状态：关闭后教师端所有用户都看不到该按钮（不是针对某个套餐隐藏）
- 排序号用于控制功能按钮的显示顺序（如菜单下拉中的排序）
- 可删除功能按钮

### 3.6 教师管理（教师端）

#### 3.6.1 教师账号列表

在原有教师管理功能基础上，新增以下展示和操作：

| 新增字段   | 说明                     |
| ---------- | ------------------------ |
| 已消耗额度 | 该教师已使用的额度点数   |
| 可用额度   | 该教师当前剩余的额度点数 |

**点数明细**：支持点击查看点数消耗的详细记录。

#### 3.6.2 新增/编辑教师

新增**额度**字段：

- 表示教师当前可用的额度点数
- 随教师使用接口逐渐减少
- 修改额度相当于重置该教师的可用额度

### 3.7 套餐管理（教师端）

#### 3.7.1 套餐状态查看

顶部展示当前套餐信息及有效期。

#### 3.7.2 点数统计

统计展示：充值点数、消费点数、扣减点数、可用点数。

#### 3.7.3 点数流水

记录学校的所有点数变动，涵盖以下类型：

| 类型 | 说明                   |
| ---- | ---------------------- |
| 充值 | 学校充值的点数         |
| 消费 | 教师使用接口消耗的点数 |
| 扣减 | 学校主动扣减的点数     |

支持筛选和导出数据。
